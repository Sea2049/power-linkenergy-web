# Powerlink Energy Official Website Design

## 1. Project Goal

Build the first official website for **www.power-linkenergy.com** using a **Cloudflare + Alibaba Cloud OSS** architecture.

The website should present:

- What products Powerlink Energy provides
- What application scenarios the company serves
- A reusable case study framework for future expansion
- A clear contact path for B2B inquiries

This website should not be a generic product catalog. It should be a **scenario-driven B2B solutions website** with product pages as supporting content.

## 2. Core Positioning

### Brand Position

Powerlink Energy is a **solution-oriented supplier of integrated power systems** for critical and industrial applications.

### Positioning Statement

Powerlink Energy delivers integrated power solutions for data centers, telecom, energy storage, and other critical applications.

### Strategic Direction

The website should emphasize:

- Scenario-based solutions instead of standalone SKUs
- One-stop integration instead of single-product trading
- Fast response, flexible MOQ, and practical support
- UPS + battery + inverter + DC/DC + cabling + monitoring integration capability

## 3. Recommended Website Approach

### Option A: Product Catalog Website

- Product-first structure
- Suitable for highly standardized inquiries
- Weak differentiation for Powerlink Energy

### Option B: Scenario Solution Website (Recommended)

- Scenario-first structure
- Product pages support solution pages
- Best fit for current materials and business direction

### Option C: Brand Trust Website

- Company, factory, certifications, and delivery capability as the main axis
- Better for a later stage when more assets are available

### Final Recommendation

Use **Option B as the main approach** and **Option A as supporting structure**.
