# Powerlink Energy README、404 与上线收尾设计说明

## 1. 本轮目标

本轮围绕正式上线前的最后一组基础交付展开，包含 3 个相互关联的目标：

- 将项目 `README.md` 统一整理为中文版本
- 补一个正式可上线的 `404.html`
- 梳理最终上线前的文件清单、发布顺序和检查步骤

这一轮属于**上线收尾包**，重点不是增加新功能，而是补齐交付面、提升正式度和降低上线执行风险。

## 2. 已确认方向

### 2.1 404 页面方向

采用 **A：品牌引导型 404 页面**。

页面需要：

- 延续官网已有视觉语言
- 保持正式官网感，而不是临时报错页
- 提供明确返回路径和转化入口

### 2.2 整体推进方式

采用 **A：上线收尾包**。

这意味着本轮会一起完成：

- README 中文统一
- 品牌型 404 页面
- 最终文件清单和发布顺序整理

## 3. 范围

本轮包含：

- `README.md` 中文化整理
- 新增正式 `404.html`
- 补强 `docs/deployment/cloudflare-oss-launch.md`
- 明确最终上线前的上传文件范围与发布顺序

本轮不包含：

- 重做全站内容架构
- 新增 CMS 或后台系统
- 调整首页、产品页、方案页主体内容
- 引入新的交互系统

## 4. 设计原则

- **上线优先**：所有输出都要直接服务上线执行
- **风格统一**：404 页面必须看起来属于当前官网，而不是独立拼接页
- **信息分层**：README 保持摘要，详细发布步骤放在独立部署文档中
- **真实可执行**：发布清单和顺序要能直接照着做

## 5. README 方案

### 5.1 角色定位

README 作为项目根文档，负责说明：

- 这个项目是什么
- 如何本地预览
- 主要有哪些页面
- Cases 如何继续维护
- 正式上线文档去哪里看

README 不承担全部部署细节，以免过长、过杂。

### 5.2 结构建议

README 调整为全中文结构，建议保留以下章节：

- 项目简介
- 技术栈
- 本地预览
- 主要页面路由
- 部署说明
- Cases 维护说明
- 上线包摘要
- 注意事项

### 5.3 文案要求

- 全文统一为中文
- 保留必要的命令、路径、URL 原样展示
- 表述偏交付文档风格，避免口语化

## 6. 404 页面方案

### 6.1 页面定位

404 页面不是简单错误提示，而是正式官网的兜底入口页。

核心任务：

- 说明页面不存在或路径已调整
- 给出清晰去路
- 保持品牌信任感

### 6.2 页面结构

页面建议使用现有官网组件体系，结构如下：

- 站点 Header
- 404 Hero 区
- 三个引导按钮
- 简短辅助说明
- 站点 Footer

### 6.3 Hero 内容建议

- 主标题：`404`
- 副标题：页面不存在、链接已更新，或当前地址不可用
- 正文：引导用户返回首页、查看 Solutions 或直接联系

### 6.4 CTA 方案

建议保留 3 个明确动作：

- 返回首页
- 查看 Solutions
- 联系我们

这样既能承接误访问流量，也能保留业务转化路径。

### 6.5 实现要求

- 新建根目录 `404.html`
- 继续复用已有 `assets/css/site.css` 和 `assets/js/site.js`
- 尽量复用统一 header/footer 与按钮样式
- 仅在必要时增加少量 404 专属样式

## 7. 上线文件清单方案

### 7.1 必须上传的文件

上线到 OSS 时，需要上传项目根目录中的正式静态站点文件，包括：

- `index.html`
- `404.html`
- `about/`
- `contact/`
- `products/`
- `solutions/`
- `cases/`
- `assets/`
- `downloads/` 中被页面引用的文件
- `robots.txt`
- `sitemap.xml`

### 7.2 不需要上传的文件

以下内容不属于正式站点文件：

- `docs/superpowers/`
- `tests/`
- 本地 brainstorming 或协作中间材料

### 7.3 文档分工

- `README.md`：项目摘要和维护入口
- `docs/deployment/cloudflare-oss-launch.md`：完整上线执行文档

## 8. 发布顺序方案

建议把最终发布顺序写成固定步骤，降低漏项风险。

### 第 1 步：本地检查

至少检查：

- `/`
- `/cases/`
- `/cases/small-data-center-backup-power/`
- `/products/`
- `/solutions/`
- `/404.html`

### 第 2 步：上传 OSS

- 上传正式静态文件
- 保持目录结构不变

### 第 3 步：配置 OSS 静态网站

- 默认首页指向 `index.html`
- 错误页指向 `404.html`

### 第 4 步：配置 Cloudflare

- 域名接入
- `www.power-linkenergy.com` 指向回源地址
- 打开代理
- 启用 HTTPS
- 根据需要开启缓存和压缩

### 第 5 步：清缓存与线上抽查

- 清理 Cloudflare 缓存
- 线上抽查核心页面、404 页面和资源加载状态

- README 已统一为中文
- 404 页面风格与官网一致
- 404 页面具备明确引导按钮
- 部署文档已包含最终上传文件清单
- 部署文档已包含明确发布顺序

## 11. 风险与处理

### 风险 1：README 过长

处理：

- 让 README 保持摘要
- 把详细流程留在独立部署文档

### 风险 2：404 页面与主站风格脱节

处理：

- 优先复用现有 header/footer、按钮和配色

### 风险 3：上线步骤遗漏

处理：

- 把发布顺序写成明确步骤
- 把上传范围和不上传范围都写清楚
